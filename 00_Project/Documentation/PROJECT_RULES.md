# PROJECT RULES

Version: 1.0

Status: Active

Owner: Ahmed Ali

Project: AI Engineer Blueprint

---

# Purpose

This document defines the rules that govern the entire AI Engineer Blueprint project.

These rules ensure consistency, maintainability, and professional engineering practices throughout the learning journey.

Every lesson, project, document, and decision must follow these rules.

---

# Rule 1 — Never Skip Fundamentals

Fundamentals are mandatory.

No advanced topic should be studied before mastering its prerequisites.

Learning speed is never more important than understanding.

---

# Rule 2 — Learn by Building

Every topic must produce practical code.

Theory without implementation is considered incomplete.

Each module ends with one or more projects.

---

# Rule 3 — Documentation First

Before adding complexity:

- Understand the problem.
- Document the solution.
- Then implement it.

Documentation is a core part of engineering.

---

# Rule 4 — Git Is Mandatory

Every meaningful milestone must be committed.

Standard workflow:

git add .

git commit

git push

No important work should remain only on the local machine.

---

# Rule 5 — One Technology at a Time

Avoid learning multiple major technologies simultaneously.

Master one concept before introducing another.

---

# Rule 6 — Understand Before Memorizing

Never memorize code blindly.

Always understand:

- What it does.
- Why it works.
- When to use it.
- When not to use it.

---

# Rule 7 — Clean Code

Every piece of code should be:

- Readable
- Simple
- Maintainable
- Beginner-friendly

Avoid unnecessary complexity.

---

# Rule 8 — Finish What You Start

Incomplete projects create incomplete understanding.

Every project must reach a usable state before moving on.

---

# Rule 9 — Portfolio Starts on Day One

Every serious project belongs on GitHub.

The portfolio is built continuously throughout the journey.

---

# Rule 10 — Ask "Why?"

Every lesson should answer:

Why does this exist?

Why is this solution used?

Why is it better?

What alternatives exist?

---

# Rule 11 — Learn Like an Engineer

The objective is not learning syntax.

The objective is learning how engineers think.

---

# Rule 12 — Practice Every Day

Consistency beats intensity.

Small daily progress compounds over time.

---

# Rule 13 — Keep Improving

Documentation is a living system.

Projects evolve.

Knowledge evolves.

The repository evolves.

Continuous improvement is expected.

---

# Rule 14 — AI Is a Mentor, Not a Replacement

AI assists learning.

The student remains responsible for:

- Thinking
- Practicing
- Solving problems
- Building projects

---

# Rule 15 — Long-Term Thinking

Every decision should support the final objective:

Becoming a professional AI Engineer capable of building production-ready AI systems.

---

# Definition of Success

This project is successful when the student can:

- Build real software.
- Build AI applications.
- Work professionally with Git.
- Understand software engineering.
- Build a strong portfolio.
- Pass technical interviews.
- Obtain an AI Engineer position.
